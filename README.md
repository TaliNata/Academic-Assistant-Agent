# Academic Assistant Agent

**Telegram-бот с двумя AI-агентами для работы с научными текстами**

Простой агент, работающий по фиксированным правилам через промпт-инжиниринг — без RAG, без векторных баз, без сложной логики принятия решений.

---

## Задача

Автоматизировать рутинные этапы подготовки научных статей к публикации:
- Первичная проверка на соответствие требованиям журнала
- Перевод аннотаций на академический английский

## Что умеет бот

### Агент-рецензент
Принимает текст статьи и проверяет:
- **Структура** — заголовок, авторы, аннотация, ключевые слова, разделы (введение → методология → результаты → выводы), список литературы
- **Оформление** — стиль цитирования, нумерация, подписи к таблицам/рисункам
- **Содержание** — цель исследования, описание методологии, соответствие выводов целям
- **Формальные требования** — объём, единообразие языка

Выдаёт структурированный чек-лист с оценками ✅ / ⚠️ / ❌ и итоговой рекомендацией (принять / доработать / отклонить).

###  Агент-переводчик
Переводит русскоязычную аннотацию на академический английский:
- Сохраняет точность и научный стиль
- Использует устоявшуюся предметную терминологию
- Выдаёт заметки по неочевидным терминологическим решениям

## Стек технологий

| Компонент | Технология | Зачем |
|-----------|-----------|-------|
| Язык | Python 3.11+ | Основа проекта |
| Telegram API | aiogram 3 | Асинхронный бот-фреймворк |
| LLM API | OpenRouter + OpenAI SDK | Доступ к множеству моделей через один API |
| Валидация | Pydantic | Типизация конфигурации и данных |
| Конфигурация | pydantic-settings + dotenv | Управление секретами через .env |

## Ключевые технические решения

**Промпт-инжиниринг как основной инструмент.** Поведение каждого агента полностью определяется системным промптом — детальной инструкцией с форматом ответа, правилами и примерами. Никакой дополнительной логики обработки результата.

**OpenRouter вместо прямого API.** Один ключ, один эндпоинт, совместимость с OpenAI SDK — и доступ к моделям Google, Anthropic, OpenAI, DeepSeek. Пользователь может сменить модель на лету через `/model`.

**Минимальная, но достаточная валидация.** Pydantic для конфигурации, проверка длины текста перед отправкой в LLM, разбиение длинных ответов по границам строк для Telegram.

## Пример работы

**Рецензент** — пользователь отправляет текст статьи, бот возвращает:

```
### Общая оценка
Статья содержит основные структурные элементы, однако имеет
ряд замечаний по оформлению и полноте содержания.

### Чек-лист
| # | Категория              | Статус | Комментарий                    |
|---|------------------------|--------|--------------------------------|
| 1 | Заголовок и авторы     | ✅     | Присутствуют                   |
| 2 | Аннотация              | ❌     | Отсутствует                    |
| 3 | Ключевые слова         | ❌     | Отсутствуют                    |
| 4 | Структура разделов     | ⚠️     | Нет обзора литературы          |
| 5 | Список литературы      | ❌     | Отсутствует                    |
| 6 | Цель исследования      | ✅     | Сформулирована                 |
| 7 | Методология            | ⚠️     | Описана кратко                 |

### Рекомендация
Доработать (существенно) — необходимо добавить аннотацию,
ключевые слова, обзор литературы и список источников.
```

## Как запустить

1. Создать бота через [@BotFather](https://t.me/BotFather)
2. Получить API-ключ на [openrouter.ai](https://openrouter.ai)
3. Заполнить `.env` (по шаблону `.env.example`)
4. `pip install -r requirements.txt`
5. `python bot.py`
